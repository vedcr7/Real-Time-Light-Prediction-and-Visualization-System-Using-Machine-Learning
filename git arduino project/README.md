# Real-Time Light Prediction and Visualization System Using Machine Learning

A hardware-software pipeline that reads light intensity from an LDR sensor via Arduino, classifies the environment as **Light** or **Dark** using a Random Forest Classifier, and displays live predictions through a Tkinter GUI with embedded Matplotlib scatter plots.

> Based on the research paper: *"Real-Time Light Prediction and Visualization System Using Machine Learning"* — Vedaansh Gupta, Graphic Era University, Dehradun, India.

---

## Problem Statement

Static threshold-based light monitoring systems cannot adapt to varying environmental conditions or leverage historical trends. This project replaces fixed thresholds with a trained ML model that learns temporal patterns from sensor data, enabling smarter and more adaptive light-state classification.

---

## Features

- Live LDR sensor reading via Arduino Uno over serial communication
- Timestamp-based feature engineering (hour, day, month, elapsed time)
- Random Forest Classifier with ~93% real-world accuracy
- Real-time prediction log with colour-coded Light / Dark entries
- Animated scatter plot that updates every 50 ms without GUI freezing
- Clean separation of concerns: data logging, training, and inference are independent scripts
- Modular Python codebase with shared utilities

## Hardware Requirements

| Component | Details |
|-----------|---------|
| Arduino Uno | Any revision |
| LDR (Light Dependent Resistor) | GL5528 or equivalent |
| Resistor | 10 kΩ (pull-down for voltage divider) |
| USB Cable | Type-A to Type-B (standard Arduino cable) |
| Jumper Wires | Male-to-male |
| Breadboard | Half-size or larger |

### Wiring Diagram

```
5V ──────┬──── LDR ────┬──── A0 (Arduino)
         │             │
        GND          10kΩ
                       │
                      GND
```

The voltage divider produces a voltage proportional to light intensity. The Arduino ADC maps this to 0–1023.

## Setup & Usage

### Step 1 — Upload the Arduino Sketch

1. Open `arduino/ldr_sensor.ino` in the Arduino IDE.
2. Select your board: **Tools → Board → Arduino Uno**.
3. Select the correct port: **Tools → Port → COMx** (Windows) or **/dev/ttyUSBx** (Linux/macOS).
4. Click **Upload**.
5. Open the Serial Monitor (9600 baud) to verify output like:
   ```
   timestamp_ms,ldr_value
   502,834
   1004,831
   1506,210
   ```

### Step 2 — Collect Training Data

```bash
python python/data_logger.py --port COM3 --output data/my_data.csv
```

Let it run for at least 30–60 minutes across both light and dark conditions to capture sufficient variance. Press **Ctrl+C** to stop.

> **No hardware?** Use the included sample dataset:
> ```bash
> python generate_sample_data.py
> ```

### Step 3 — Train the Model

```bash
python python/train_model.py --data data/sample_data.csv
```

This will:
- Print a classification report (accuracy, precision, recall, F1)
- Save a confusion matrix + feature importance chart to `assets/model_evaluation.png`
- Save the trained model to `python/model.pkl`

Sample output:
```
[train] 5-fold CV accuracy: 0.9312 ± 0.0184
[train] Test accuracy: 0.9320 (93.20%)

              precision    recall  f1-score   support
        Dark       0.91      0.95      0.93        82
       Light       0.97      0.94      0.96       158
    accuracy                           0.95       240
```

### Step 4 — Run the Real-Time GUI

```bash
python python/realtime_gui.py --port COM3
```

The GUI will:
- Connect to the Arduino automatically
- Load `python/model.pkl`
- Display live predictions in the scatter plot
- Log every reading with timestamp, LDR value, and predicted state

## Model Details

| Parameter | Value |
|-----------|-------|
| Algorithm | Random Forest Classifier |
| Estimators | 100 decision trees |
| Features | `ldr_value`, `hour`, `day`, `month`, `day_of_week`, `elapsed_time` |
| Target | `1` = Light, `0` = Dark (threshold: LDR ≥ 512) |
| Train/Test split | 80% / 20% (stratified) |
| Cross-validation | 5-fold stratified |
| Reported accuracy | ~93.20% |

### Why Random Forest?

- Handles non-linear relationships between time-of-day and sensor readings
- Resistant to overfitting via ensemble averaging across 100 trees
- Provides feature importance rankings for interpretability
- No feature scaling required

## Known Limitations

1. **Sensor dependency** — Model accuracy is bounded by LDR sensor quality. Dirty optics or loose connections introduce noise.
2. **Computational overhead** — Continuous serial reads and GUI redraws may lag on low-spec machines when `MAX_PLOT_POINTS` is increased.
3. **Fixed threshold label** — Training labels are derived from a static LDR threshold (512). Environments with unusual ambient baselines may need threshold recalibration in `utils.py`.
